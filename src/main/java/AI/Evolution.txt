
AI发展三阶段总结（贴合个人开发体验）

一、中文版本

阶段1：纯被动对话时代（2024年底—2025年初）

核心体感：仅提供聊天框交互，通过输入Prompt获取答案，AI仅能答疑、提供代码片段思路，无法操作本地项目或开发工具。

代表产品：DeepSeek、Claude 3、GPT-4早期版本

核心能力：仅支持文字问答，不能读取本地项目文件、无法嵌入IDE、不能修改工程代码、不具备联网查询能力。

角色定位：人主导开发，AI仅作为问答工具，需手动复制粘贴代码片段，AI仅提供修改建议，不参与实际工程操作。

本质：AI是单纯的“问答助手”，仅解决基础咨询和简单代码建议需求。

阶段2：IDE内嵌智能体 · 局部改代码（2025年初—2025年中）

核心体感：AI深度嵌入IDE，可直接读取本地完整Project，理解项目目录、依赖配置和全链路代码，能自主定位bug、批量修改多个文件、补充接口、调整依赖。

代表产品：Cursor、Claude Code、各类IDE插件型AI Agent

核心突破：AI具备完整工程上下文理解能力，可实现本地文件读写、跨文件修改，能贴合具体业务逻辑进行代码操作。

幕后支撑：MCP（模型上下文协议）作为底层技术，无需用户直接接触，支撑AI与本地文件、IDE的连接与交互。

角色定位：人依然主导开发，AI作为“高级结对编程伙伴”，需人工指挥操作方向，AI执行具体修改，修改后由人工验收确认。

本质：AI从“问答工具”升级为“辅助开发工具”，减少人工重复编码工作，但核心决策仍由人掌控。

阶段3：AI主导 · 端到端自主开发（2025年底—2026年至今）

核心体感：仅提供需求文档（或一句话需求），AI自主完成技术方案设计、架构选型，经人工Review调整后，AI生成全量代码、编写所有测试用例，全程无需人工编写一行代码。

相关概念归属：OpenClaw（开源本地自主Agent，具备文件读取、命令执行、浏览器控制、长记忆、联网能力）、Web Coding / Vibe Coding（氛围编程，通过自然语言描述需求，AI自主完成架构、编码、部署全流程）均属于此阶段。

核心能力：需求分析→技术方案设计→全量代码生成→单元/集成测试→bug自动修复→文档生成，实现端到端自主开发；支持联网查询最新技术文档、框架版本，适配企业级开发需求。

角色定位：AI主导开发全流程，人仅负责需求输出、技术方案Review、最终验收和部署把关，无需参与具体编码和测试工作。

行业共识名称：Vibe Coding / Spec Coding（规范驱动开发）、Agentic Engineering（智能体工程）、End-to-End AI Development（端到端AI开发）。

体感特点：变化为流程级、隐性渐进式，无“IDE内嵌”那样的直观功能冲击，核心是“人主导”到“AI主导”的开发范式反转。

核心总结

整体演进路径（完全贴合个人体验）：纯被动对话（AI问答工具）→ IDE内嵌智能体（AI辅助开发）→ AI主导端到端自主开发（OpenClaw、Web Coding归属此阶段），本质是AI从“辅助工具”向“自主开发者”的升级，开发角色从“人写代码、AI辅助”反转为“AI写代码、人评审”。

二、英文版本

Stage 1: Pure Passive Conversation Era (Late 2024 — Early 2025)

Core Experience: Only chat box interaction is provided. Answers are obtained by inputting Prompts. AI can only answer questions, provide ideas for code snippets, and cannot operate local projects or development tools.

Representative Products: DeepSeek, Claude 3, early versions of GPT-4

Core Capabilities: Only supports text Q&A; cannot read local project files, cannot be embedded in IDE, cannot modify project code, and does not have internet search capabilities.

Role Positioning: Humans lead development. AI only serves as a Q&A tool. Code snippets need to be copied and pasted manually, and AI only provides modification suggestions without participating in actual engineering operations.

Essence: AI is a simple "Q&A assistant" that only addresses basic consulting and simple code suggestion needs.

Stage 2: IDE-Embedded Agent · Partial Code Modification (Early 2025 — Mid 2025)

Core Experience: AI is deeply embedded in IDE, which can directly read the complete local Project, understand project directories, dependency configurations and full-link code, and can independently locate bugs, batch modify multiple files, supplement interfaces, and adjust dependencies.

Representative Products: Cursor, Claude Code, various IDE plug-in-based AI Agents

Core Breakthrough: AI has the ability to understand the complete engineering context, can realize local file reading and writing, cross-file modification, and can perform code operations in line with specific business logic.

Underlying Support: MCP (Model Context Protocol) serves as the underlying technology, which does not require direct user contact and supports the connection and interaction between AI and local files/IDEs.

Role Positioning: Humans still lead development. AI serves as an "advanced pair programming partner". Manual guidance on operation direction is required, AI executes specific modifications, and the modifications are verified and confirmed manually.

Essence: AI has evolved from a "Q&A tool" to an "auxiliary development tool", reducing manual repetitive coding work, but core decisions are still controlled by humans.

Stage 3: AI-Led · End-to-End Autonomous Development (Late 2025 — Present 2026)

Core Experience: Only provide requirement documents (or one-sentence requirements). AI independently completes technical scheme design and architecture selection. After manual review and adjustment, AI generates full-scale code and writes all test cases, without manual writing of a single line of code throughout the process.

Attribution of Related Concepts: OpenClaw (an open-source local autonomous Agent with file reading, command execution, browser control, long-term memory, and internet capabilities), Web Coding / Vibe Coding (atmospheric programming, where AI independently completes the entire process of architecture, coding, and deployment through natural language description of requirements) all belong to this stage.

Core Capabilities: Requirement analysis → technical scheme design → full-scale code generation → unit/integration testing → automatic bug fixing → document generation, realizing end-to-end autonomous development; supporting internet search for the latest technical documents and framework versions, adapting to enterprise-level development needs.

Role Positioning: AI leads the entire development process. Humans are only responsible for requirement output, technical scheme review, final acceptance, and deployment supervision, without participating in specific coding and testing work.

Industry Consensus Names: Vibe Coding / Spec Coding (Specification-Driven Development), Agentic Engineering, End-to-End AI Development.

Experience Characteristics: The change is process-level and implicitly progressive, without the intuitive functional impact like "IDE embedding". The core is the reversal of the development paradigm from "human-led" to "AI-led".

Core Summary

Overall Evolution Path (Fully Aligned with Personal Experience): Pure Passive Conversation (AI as Q&A Tool) → IDE-Embedded Agent (AI-Assisted Development) → AI-Led End-to-End Autonomous Development (OpenClaw and Web Coding belong to this stage). The essence is the upgrade of AI from an "auxiliary tool" to an "autonomous developer", and the development role is reversed from "humans write code and AI assists" to "AI writes code and humans review".
